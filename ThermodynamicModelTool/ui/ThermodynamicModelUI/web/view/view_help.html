Thermodynamic Model Tool User Documentation

Use this model tool to as a way to quickly specify your thermodynamic transcription model.
All you need to do is to input your data to the forms accessible through the panel on the left.

Theory

What we mean by a thermodynamic model of gene expression is a mathematical model of dependency of protein concentrations in time expressed as a collection of ordinary delay differential equations that makes use of the Shea-Ackers kinetic to model gene transcription.

The Shea-Ackers kinetic is what makes it a thermodynamic model since it is justified by assuming that due to the time scale separation between the slow transcription and relatively fast binding of transcription factors the latter can be viewed as being in a quasi-steady state regarding the former. Therefore using thermodynamic arguments in deriving a mathematical model to describe the transcription rate is valid.

One of the intermediary results of the Shea-Ackers kinetic is called fractional occupancy, giving the fraction of promotors in a large ansamble of individual systems that are bound by RNAP. This is the factor that detemines the transcription rate. For this reason the thermodynamic model is often called a fractional occupancy model.

The simulation of the model comprises the numeric integration of the system of DDEs. At each integration step, the fractional occupancy is reevaluated using the current protein concentrations.

List of relevant references to literature is provided and the theory is reviewed in more detail in the report by the Autumn 2013 group.

Thermodynamic Model Tool

We choose to represent a thermodynamic model as a set of Constants, Parameters, Species, Reactions and Rate Laws. To specify a model, visit each of the sections in the left menu in turn and add these components.

The initial concentration of Species is specified through the Param Eval Tool.

The difference between Constants and Parameters is that values of constants are specified here while parameter values are specified through a Param Eval Tool. Conceptually, constants are given and do not change during the experiment, while the values of parameters depend on the experimental setup.

Computational Biology Workspace (CBW)

This section is based on the CWB documantation in Slovenian language.

The Computational Biology Workspace is built upon the NetBeans Platform. For the best compatibility it is necessary to have installed version 7.4. The application is written to operate on Microsoft Windows and has compatibility problems on other platforms (with JavaFX libraries, which are located in the ./Release/modules/bin).

Installation

Download In NetBeans select File->Open select the folder where you downloaded CBW (it will have a puzzle icon), check the Open required projects checkmark and confirm the dialog by clicking OK. Click the Run button (Green arrow) on the top toolbar

Using the application

The basic two applications are the Palettei> which contains the available tools and Core window where the available widgets are placed by dragging them from the Pallete using a mouse and interconnected. Tools in the Core window are treated as nodes which are interconnected by oriented edges. The orientation of the edges indicates the flow of data. Tools can be connected to each other by holding Ctrl key and clicking the left mouse button.

Although the application allows interconecting all tools with any other, you have to be careful, because tools are compatible only with some predecessors. In general a a PlotTool tool gets data from the EngineTool, which gets data from a ParamEvalTool which gets data from a ModelTool.

Right clicking on the node displays the popup menu that we offer to delete the node or display a window with additional configuration.

The four CBW tools that are used for specifying and evaluating the thermodynamic model are

  • ThermodynamicModelTool
  • ThermodynamicParamEvalTool
  • ThermodynamicSimulationEngine
  • plot

Notes

As of December 2013, this module is also compatible with ***deterministic euler integrator*** provided that delay terms are not used. It might be possible to convert some DDEs to ODEs in a way described at the Copasi website.

There is a gap between how the names are displayed (with \(\TeX\) math formatting) and how they are represented internally. This can cause problems if the user is not aware of the implications. The internal representation for symbols is the string entered by the user. Therefore, even if two constant names are rendered the same, they will be considered different constants if the syntax used to specify them differs.
